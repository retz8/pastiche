# Task 8.1 — Repo Restructure (`core/ + adapters/`) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move pastiche's source concepts into the locked `core/ + adapters/` layout and repoint every build/config reference, with **zero content changes**, keeping the existing build producing the identical `dist-plugin/` artifact.

**Architecture:** Pure `git mv` relocations plus path-reference fixes. `agents/`, `skills/`, `templates/` move under `core/`; the extractor (TS) and lint (Rust) move under `core/tools/`; `adapters/`, `scripts/build.ts`, and `scripts/field-test/` stay at their current roots. Each move-unit is paired with its reference updates so the unit is independently verifiable; the full build is the final gate. This task does **not** touch the distribution layout (`dist/claude-code/`, `marketplace.json`, version mechanism, `--platform` flag) — that is task 8.2.

**Tech Stack:** Node `node --test` + `tsx` (extractor tests), `tsup` (extractor bundle), Cargo (Rust lint), a `tsx` build orchestrator (`scripts/build.ts`).

**Confirmed NON-issues (do not touch):**
- No skill/agent/adapter body hardcodes a source path (they invoke `extract-fact`/`pastiche-lint` by name off the built `bin/` PATH).
- `extract-fact-ts.ts` imports only `node:fs`, `node:path`, `yaml`, `ts-morph` — no sibling imports.
- The extractor test resolves fixtures and the script-under-test via `import.meta.url`-relative URLs (`./extract-fact-ts.ts`, `./__fixtures__/source-walk`), so moving the test + `__fixtures__/` **together with** the source preserves both.

From `scripts/` (1 level below root) `../node_modules` = root `node_modules`; from `core/tools/extract-fact/` (3 levels below root) it would resolve to the non-existent `core/tools/node_modules`. This hop must change to `../../../node_modules`. Every `runMain`-based e2e test depends on it.
- Rust `src/` and `tests/` contain no repo-relative path literals.
- No `vitest`/`jest` config exists.

**Real command names (the task brief's `lint:rust`/`vitest` were wrong):** test runner is `npm test` → `node --test`; Rust gate is `npm run test:lint` → `cargo test`.
