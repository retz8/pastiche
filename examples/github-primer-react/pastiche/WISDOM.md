- [GENERAL] Never use raw hex, rgb, or px values; always reference Primer design tokens (`--fgColor-*`, `--bgColor-*`, `--borderColor-*`, `--shadow-*`).
- [GENERAL] No arbitrary gap or padding values; use Stack's gap scale (`none`, `tight`, `condensed`, `cozy`, `normal`, `spacious`) or Primer spacing tokens.
- [GENERAL] No arbitrary media queries; use Primer's viewport ranges (narrow <768px, regular ≥768px, wide ≥1400px) or named breakpoints (xsmall–xxlarge).
- [GENERAL] Contrast ratios (4.5:1 text, 3:1 non-text), focus-visible on all interactive elements, semantic HTML by default, `aria-label` on every icon-only control.
- [GENERAL] `Box` is not exported from `@primer/react`. For generic containers needing Primer token styling, use a plain HTML element (`div`, `section`, etc.) with `style` referencing Primer CSS custom properties, or use `Stack` when flex layout with gap is needed.
- [GENERAL] Always wrap rendered text in the appropriate typographic component — `Text` for body and inline text, `Heading` (with the correct `as` prop for h1–h6) for heading-level text. Never place raw strings directly inside layout or interactive components; the typographic component establishes its own context (size, weight, color), preventing unintended inheritance.
- [Button] Never render more than one variant="primary" Button in a single button group or page region. Primary is the single highest-priority action.
- [Dialog] Before using size="xlarge" (640px), consider whether the content belongs on a separate page. Dialogs are for transient content, not full workflows.
- [IconButton] Always provide aria-label describing the button's action. IconButton has no visible text — the label is the only accessible name.
- [NavList] Never substitute TreeView for NavList when building navigation. TreeView is for hierarchical data display, not page navigation — it uses different ARIA roles and keyboard patterns.
- [FormControl] Never disable or hide the submit button, even when the form is invalid. Disabled buttons break keyboard navigation and leave users without feedback on what went wrong.
- [Stack] When composing a page's top-level sections (page header, filters, content list, pagination), wrap them in a vertical Stack with a named gap (typically `normal` or `spacious`). Never rely on components' intrinsic margins alone for inter-section spacing — Primer components render flush by default.
- [Heading,Text] The combined font-size scale across both components is: Heading large (2rem) → Heading medium (1.25rem) → Heading small / Text large (1rem) → Text medium (0.875rem) → Text small (0.75rem). Heading variant="small" and Text size="large" render at the same font size; choose sizes from this unified scale to maintain clear visual hierarchy.
- [Heading,Text,Stack] Heading and Text carry inherent line-height (e.g. Text medium has `line-height: 1.5`) that already provides vertical breathing room. When composing a heading + description pair inside a Stack, use `gap="none"` — the line-height provides sufficient visual separation without an explicit gap.
- [Stack] In a horizontal Stack laying out text of differing font sizes on one line (e.g. a semibold name beside a smaller, lighter timestamp/byline), use `align="baseline"` rather than `align="center"` — centering mixed-size text reads as misaligned, while a shared baseline sits the glyphs on one line. This holds for bare text and icons only: if the row also contains a boxed atom (Label, Button, Token, CounterLabel), use `align="center"` instead — boxed atoms have no shared text baseline and sag below the line when baseline-aligned.
- [Stack] In a horizontal Stack pairing a leading icon/visual with a vertical title + description block, use `align="start"` so the icon aligns with the title's top line — `align="center"` centers the icon against the full two-line block, leaving it floating between the title and description.
- [Stack] In a horizontal Stack pairing a leading icon/visual with single-line text (e.g. a status icon beside a text label), use `align="baseline"` so the icon sits on the text baseline — `align="center"` optically floats a small icon against the text's cap height. This holds when the row is only an icon and text: if the row also contains a boxed atom (Label, Button, Token, CounterLabel), use `align="center"` instead — boxed atoms have no shared text baseline and sag below the line when baseline-aligned.
- [Stack] In a horizontal Stack with justify="space-between", every direct child is distributed — the first pins to the start, the last to the end, and any middle child strands in the dead space, reading as floating. Keep exactly two children (a leading and a trailing group), wrapping related trailing items in their own nested Stack so the split falls only between the two clusters.
- [SplitPageLayout.Sidebar] Sidebar does not pad its own content — a `padding` prop on Sidebar has no effect. To inset sidebar content, put the padding on the inner wrapper (e.g. the `Stack` that wraps the items via its `padding`/`paddingBlock`/`paddingInline` props), not on Sidebar.
- [ActionMenu.Button] ActionMenu.Button renders its own trailing dropdown caret automatically. Do not pass an explicit caret via `trailingVisual` (e.g. `trailingVisual={TriangleDownIcon}`) — it produces a duplicate, doubled arrow.
